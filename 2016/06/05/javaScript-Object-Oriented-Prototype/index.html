JS中使用prototype来实现面向对象编程

發表於 2016-06-05   |  

ECMAScript 支持面向对象编程,但在ES6之前不使用类或者接口.对象在代码执行的过程中创建和增加更多的方法.在没有类的情况下可以采用下面几种模式来创建对象:

工厂模式

  • 使用了简单的函数来创建对象,并且为对象增加属性和方法,然后返回对象.(抽象创建具体对象的过程)

优缺点:

  • 优点:不创建多个相似的对象,减少重复的代码
  • 缺点:不能解决对象识别的问题(即对象没有特定的构造函数,不能通过instanceof来
    识别 他是否是某个对象的实例).

构造函数模式

可以自定义构造函数的模式:

执行过程:

  1. 创建一个新对象
  2. 将构造函数的作用域赋给新对象.(this指向新对象)
  3. 执行构造函数中的代码(添加属性(值和方法));
  4. 返回新建的对象.

优点和缺点:

  • 优点:相比于工厂模式,可以用instanceof或者constuctor识别对象的类型.

  • 缺点:完成相同任务的方法,创建了多次实例,不能代码共用.

原型模式

理解原型对象

在对象的prototype中查找属性与方法就是一次搜索,所以对对象的值得修改能够立即从实例上反应出来.

优点&缺点:

  • 优点:所有方法属性都可以共享,提高了代码重用率
  • 缺点:所有属性被所有实例共享,一个实例修改所有,所有实例获得的属性都会改变,需要结合构造函数来解决这个问题.

  • 使用Object.keys获取所有可枚举的的属性
  • 使用Object.getOwnPropertyNames()获取所有实例的属性(包括不可枚举的)

ES6中的实现

ES6中 定义面向对象的类,只需要以class开头 然后是类型名,不再需要使用function关键字,只需方法名和圆括号.在constructor方法中定义的是这个类的构造函数.所有的属性和方法都会被添加到原型上.sayName和getOlder方法都是定义在Person.prototype上的.
